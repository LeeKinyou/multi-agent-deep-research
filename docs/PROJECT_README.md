# MultiAgentDeepResearch —— 多智能体分析系统 项目说明书

***

## 1. 项目背景

### 1.1 行业背景

在信息爆炸的时代，企业和研究人员面临着海量信息的筛选与分析挑战。传统的研究报告生成方式依赖人工收集资料、整理分析，耗时耗力且难以保证信息的全面性和时效性。随着人工智能技术的快速发展，特别是大语言模型（LLM）和多智能体（Multi-Agent）技术的成熟，自动化、智能化的研究分析系统成为可能。

### 1.2 问题陈述

当前市场存在以下痛点：

- **信息收集效率低**：人工搜索和筛选信息耗时巨大
- **分析维度单一**：难以同时从商业、技术、市场等多个角度进行交叉分析
- **报告质量不稳定**：依赖个人经验和能力，难以标准化输出
- **实时性差**：无法及时获取最新信息并更新分析结论
- **缺乏系统规划**：研究过程缺乏结构化规划，容易遗漏关键维度

### 1.3 项目意义

本项目旨在构建一个具备计划制定能力的深度研究系统，通过主编Agent的智能规划，实现从主题拆解、信息采集到多维度分析、交叉验证的全流程自动化，大幅提升研究效率和报告质量。

***

## 2. 核心目标

### 2.1 总体目标

构建一个基于多智能体协作的深度研究系统，用户只需输入研究主题，主编Agent自动制定研究计划，调度各专业Agent按计划执行信息采集、多维度分析、交叉验证，最终输出结构化的专业研究报告。

### 2.2 具体目标

| 目标编号 | 目标描述         | 关键指标                         |
| ---- | ------------ | ---------------------------- |
| G1   | 实现智能计划制定     | 主编Agent自动生成结构化研究计划           |
| G2   | 实现多Agent协作流程 | 至少4个核心Agent协同工作（主编、采集、分析、撰写） |
| G3   | 自动化信息采集      | 支持实时网络搜索和网页内容抓取              |
| G4   | 多维度分析能力      | 覆盖商业、技术、竞品等维度，支持交叉验证         |
| G5   | 结构化报告输出      | 支持Markdown和PDF格式导出           |
| G6   | 交互式Web界面     | 提供友好的对话式用户体验，支持计划确认          |
| G7   | 任务状态管理       | 支持异步任务执行和状态追踪                |

### 2.3 目标用户

- 行业研究人员
- 市场分析师
- 投资分析师
- 企业战略规划人员
- 学术研究人员
- 咨询顾问

#### 3.2.3 多智能体协作模块

| 功能编号 | 功能名称        | 功能描述                        | 优先级 |
| ---- | ----------- | --------------------------- | --- |
| F3.0 | 主编/规划 Agent | 制定研究计划，拆解任务，调度执行，监控进度       | P0  |
| F3.1 | 情报采集 Agent  | 执行网络搜索，收集相关信息               | P0  |
| F3.2 | 商业分析 Agent  | 进行SWOT分析、竞品分析、市场分析          | P0  |
| F3.3 | 技术分析 Agent  | 分析技术趋势、技术栈、技术可行性（与商业分析侧重不同） | P1  |
| F3.4 | 报告生成 Agent  | 整合分析结果，生成结构化报告              | P0  |
| F3.5 | 交叉验证 Agent  | 验证信息准确性，识别矛盾点，触发补充搜索或辩论     | P1  |
| F3.6 | 人机协同        | 关键节点暂停，等待用户确认或补充            | P1  |

**Agent团队结构说明**：

- **核心Agent（P0）**：主编规划Agent、情报采集Agent、商业分析Agent、报告生成Agent（共4个）
- **扩展Agent（P1）**：技术分析Agent、交叉验证Agent（可根据研究主题动态启用）
- **技术分析Agent与商业分析Agent的区别**：
  - 商业分析Agent：侧重市场格局、商业模式、竞争策略、SWOT分析
  - 技术分析Agent：侧重技术架构、技术趋势、技术可行性、技术风险评估

### 3.3 报告输出规范

#### 3.3.1 报告结构

```markdown
# {研究主题} 研究报告

## 1. 执行摘要
- 研究背景
- 核心发现
- 关键建议

## 2. 研究方法与数据来源
- 研究范围
- 数据来源
- 分析方法

## 3. 市场分析
- 市场规模与趋势
- 目标用户群体
- 竞争格局

## 4. 竞品分析
- 主要竞品列表
- 竞品功能对比
- 竞品优劣势分析

## 5. SWOT分析
- 优势 (Strengths)
- 劣势 (Weaknesses)
- 机会 (Opportunities)
- 威胁 (Threats)

## 6. 技术分析（如适用）
- 技术趋势
- 核心技术栈
- 技术可行性评估

## 7. 结论与建议
- 核心结论
- 行动建议
- 风险提示

## 8. 参考来源
- 信息来源列表
```

#### 3.3.2 输出格式

- **Markdown格式**：默认输出，便于在线预览和编辑
- **PDF格式**：可选导出，适合正式汇报和存档

goal: 对不同Agent的分析结论进行交叉验证，识别信息矛盾点，推动补充研究，确保报告结论的准确性和一致性
backstory: 你是一位严谨的研究质量把控专家，擅长发现分析中的逻辑漏洞和信息冲突，能够通过多源验证确保研究结论的可靠性
validation_mechanism:
  - 对比商业分析与技术分析的结论差异
  - 识别信息来源的矛盾点
  - 触发补充搜索验证
  - 协调相关Agent进行"辩论"澄清
  - 向主编汇报验证结果
skills:
  - 逻辑分析
  - 矛盾识别
  - 多源验证
  - 协调沟通
tools:
  - 对比分析
  - 补充搜索

#### 4.3.2 任务流程设计

**主流程：计划驱动的研究执行**

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   开始      │────→│  计划制定   │────→│  计划确认   │
│  输入主题   │     │  主编Agent  │     │  用户/自动  │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                               │
                                               ↓
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   输出      │←────│  报告生成   │←────│  情报采集   │
│  研究报告   │     │  任务       │     │  任务       │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                               │
                                               ↓
                                        ┌─────────────┐
                                        │  商业分析   │
                                        │  任务       │
                                        └─────────────┘
```

**可选分支流程：交叉验证循环**

```
                    ┌─────────────────┐
                    │   商业分析      │
                    │  Agent输出      │
                    └────────┬────────┘
                             │
                             ↓
┌─────────────┐     ┌─────────────────┐     ┌─────────────┐
│  补充搜索   │←────│   交叉验证      │←────│  技术分析   │
│  验证差异   │     │  主编Agent      │     │  Agent输出  │
└──────┬──────┘     └────────┬────────┘     └─────────────┘
       │                     │
       └─────────────────────┘
                             │
                             ↓
                    ┌─────────────────┐
                    │  结论一致/      │
                    │  差异已解决     │
                    └─────────────────┘
```

**交叉验证触发条件**：

1. 商业分析Agent和技术分析Agent的结论存在明显矛盾
2. 同一信息在不同来源中存在冲突
3. 主编Agent识别出逻辑漏洞

**验证机制**：

- 主编Agent对比各方结论
- 自动触发补充搜索验证矛盾点
- 必要时组织相关Agent进行"辩论"澄清
- 最终由主编Agent裁定或标记待确认

### 4.4 数据模型设计

#### 4.4.1 任务表 (tasks)

```sql
CREATE TABLE tasks (
    id TEXT PRIMARY KEY,
    topic TEXT NOT NULL,
    status TEXT NOT NULL, -- planning, pending, running, completed, failed, cancelled
    progress INTEGER DEFAULT 0,
    current_step TEXT,
    plan TEXT, -- JSON格式存储研究计划
    plan_confirmed BOOLEAN DEFAULT FALSE,
    plan_confirmation_time TIMESTAMP,
    result_path TEXT,
    error_message TEXT,
    options TEXT, -- JSON格式存储选项
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP
);
```

#### 4.4.2 研究计划表 (research\_plans)

```sql
CREATE TABLE research_plans (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id TEXT NOT NULL,
    plan_content TEXT NOT NULL, -- JSON格式
    version INTEGER DEFAULT 1,
    status TEXT, -- draft, confirmed, modified
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (task_id) REFERENCES tasks(id)
);
```

#### 4.4.3 执行日志表 (execution\_logs)

```sql
CREATE TABLE execution_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id TEXT NOT NULL,
    agent_name TEXT,
    step_name TEXT,
    log_level TEXT, -- info, warning, error
    message TEXT,
    validation_event BOOLEAN DEFAULT FALSE, -- 是否为验证相关事件
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (task_id) REFERENCES tasks(id)
);
```
